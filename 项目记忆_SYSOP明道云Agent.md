# 项目记忆 · SYSOP 明道云 Agent

> 给后续会话使用的上下文备忘：这个工作区是什么、要服务什么目标、默认怎么处理明道云任务。

## 一、项目定位

本工作区用于建设一个 **明道云自然语言操作 agent**。

目标不是维护 SOP 网站，而是让用户可以在 Codex / Claude Code 中用自然语言完成原本需要进入明道云手工处理的事情：

- 查询订单、库存、SKU、入库、出库、退换货、审批、流程状态。
- 排查为什么某条订单/库存/退换货没有按预期流转。
- 维护工作表字段、视图、关联、看板、自动化流程。
- 设计和搭建新的明道云业务能力。
- 把重复操作沉淀成可复用命令、文档和规则。

当前仓库里仍保留 SOP 帮助中心内容，这是历史流程参考，不再是本工作区的默认目标。

## 生产环境提醒

当前明道云应用是生产环境。Agent 必须非常谨慎：

- 默认只读识别，不能默认写入。
- 不清楚业务作用的字段、视图、工作表、工作流，先查结构和样例；仍不清楚就问用户。
- 执行前要理解系统：这个对象属于什么流程、会触发什么自动化、影响哪些上下游表。
- 影响订单、库存、财务、审批、客户结果的动作必须明确确认。

## 三、默认处理方式

用户说“查一下”：

1. 先识别对象：订单号、SKU、学校、批次、手机号、快递单号、出库单序号等。
2. 查相关工作表和视图。
3. 输出当前状态、证据字段、下一步建议。

用户说“改一下/维护一下”：

1. 先定位记录和字段。
2. 识别字段/视图/工作流作用。
3. 判断风险。
4. 低风险可执行；高风险必须确认。
5. 改完回查验证。

用户说“搭一个/做一个能力”：

1. 先设计数据结构和流程。
2. 明确哪些表、字段、视图、自动化要新增或改动。
3. 先搭低风险结构；发布流程或影响业务动作前确认。
4. 搭完用样例记录验证。

## 四、当前重点方向

- 把订单、库存、入库、出库、退换货相关的常用查询变成自然语言可执行。
- 建立字段/视图/流程 ID 索引，减少每次重新摸结构。
- 为库存周转、库龄、滞销、异常出入库等指标设计可落地口径。
- 把高风险动作和低风险维护动作分清，避免 agent 误改真实业务。
