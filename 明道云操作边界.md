# 明道云操作边界

> 判断 agent 能不能直接执行明道云动作。默认先保护真实业务数据，再追求自动化。

## 零、生产环境默认策略

当前明道云应用属于生产环境。默认策略：

- **只读优先**：不清楚业务作用时，只能查，不能改。
- **确认优先**：可能影响订单、库存、财务、审批、客户结果、自动化流转的动作，必须先确认。
- **识别优先**：对字段、视图、按钮、工作流节点作用不清楚时，先识别系统结构和样例记录。
- **最小变更**：能用测试记录验证，就不要先动真实记录；能小范围验证，就不要批量操作。
- **可回滚意识**：执行前要知道如何撤回或修正；不知道回滚方式的高风险动作不要执行。

## 一、可直接执行

这些动作默认低风险，可直接做，做完说明结果：

- 查询应用、工作表、字段、视图、流程结构。
- 查询少量记录样例。
- 查询订单、SKU、库存、入库、出库、退换货、快递、审批状态。
- 统计汇总、透视、生成分析口径。
- 新建或更新本地文档。
- 更新本地索引、操作日志、说明文档。

## 二、通常可执行，但要说明影响

这些动作一般不直接改变业务结果，但仍要说明会影响哪里：

- 新增辅助字段。
- 新增只读视图、筛选视图、统计视图。
- 新增说明类字段、备注类字段。
- 修改本地文档中记录的字段/视图 ID。
- 创建测试用工作表或测试记录。

## 三、执行前应先确认

这些动作可能影响多人协作或业务口径，执行前至少说明计划和影响范围：

- 修改已有字段名称、字段类型、选项。
- 修改已有视图筛选、排序、显示字段。
- 批量更新非关键字段。
- 新建自动化流程但不发布。
- 修改看板/统计口径。
- 对真实记录做小范围补充备注或辅助标记。

## 四、必须用户明确确认

这些动作涉及真实订单、财务、库存、审批、客户结果，必须用户明确授权：

- 任何 agent 不完全理解作用、但可能改变生产环境的动作。
- 删除记录、字段、视图、工作表、流程。
- 修改订单状态、出库状态、入库状态、审批状态。
- 审批通过或驳回。
- 释放库存、锁库存、调整库存数量。
- 生成退款单、入库单、出库单、移仓单、虚拟移仓单。
- 批量导入、批量修改、批量关联、批量删除。
- 发布、停用、删除自动化工作流。
- 修改工作流触发条件、筛选条件、动作节点、审批节点。
- 修改财务金额、采购成本、销售金额、退款金额。

确认话术要具体，例如：

> 我将把 X 表中筛选到的 N 条记录的 Y 字段从 A 改为 B，会影响订单状态/库存/财务。请确认是否执行。

## 五、禁止默认做

除非用户非常明确并且你已经复述影响范围，否则不要做：

- “顺手清理”真实业务数据。
- 看到异常就自动修正订单/库存/审批。
- 对不认识的工作流、字段、视图、按钮直接执行写入。
- 用猜测字段 ID 写入。
- 在没有验证样例的情况下跑批量操作。
- 为了让流程跑通而手动改业务结果。

## 六、未知对象处理规则

遇到不清楚作用的字段、视图、按钮、工作流或工作表时：

1. 先查名称、备注、字段类型、视图筛选、样例记录。
2. 查上下游关联记录。
3. 查相关工作流结构和触发条件。
4. 如果仍然不能判断作用，向用户提问，不执行写入。

提问要具体，例如：

> 我看到字段 X 会被工作流 Y 使用，但无法确认它是否控制库存释放。这个字段是否允许我修改？

## 七、验证要求

写操作后必须至少做一种验证：

- 回查被修改记录。
- 回查字段/视图/流程结构。
- 查操作影响的统计数。
- 用样例记录跑一遍流程。

验证结果要告诉用户，不能只说“已完成”。
